1. What is Chord? (Use a single sentence)
Answer: A Scalable Peer-to-peer Lookup Service for Internet
Applications that can efficiently locate the node that stores a particular data item.

2. What is the problem Chord addresses and why is it important??(Use no more than two sentences)
Answer: Load balance, decentralization, scalability, availability and flexible naming. It is important because such a system can be used by many distributed peer-to-peer applications that need to determine the node that stores a data item.

3. What is the core idea in Chord that differentiates it from prior work on DHTs?
Answer:  Chord maintains additional routing information (finger tables) to accelerate the process of lookups.
